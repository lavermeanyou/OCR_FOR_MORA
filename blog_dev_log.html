명함 OCR 필드 분류기 — 개발 과정과 트러블슈팅 기록

PaddleOCR Python Claude Code 명함 인식 MVP

1. 프로젝트 소개

명함 이미지를 업로드하면 OCR로 텍스트를 추출하고, 각 텍스트 블록을 의미 있는 필드(회사명, 이름, 직책, 전화번호 등)로 자동 분류하는 시스템을 만들고 있다.

OCR 엔진 자체를 만드는 게 아니라, PaddleOCR가 뱉어준 텍스트를 어떤 필드에 매핑할 것인가가 핵심이다. MVP 단계라 스키마는 6개 필드로 제한했다.

내부 코드명사용자 표시 (한국어)
company_name회사이름
person_name이름
job_title직책
phone_number전화번호
fax_number팩스번호
email이메일

3-2. 데이터 포맷 설계

OCR 결과와 라벨링 데이터를 JSON으로 정의했다. 두 가지 포맷을 만들었다.

OCR 출력 포맷 — PaddleOCR 결과를 그대로 저장:

{
  "image_file": "card_001.jpg",
  "ocr_engine": "PaddleOCR",
  "text_blocks": [
    {
      "text": "ABC Corporation",
      "confidence": 0.97,
      "bbox": [[50,30], [300,30], [300,70], [50,70]],
      "block_index": 0
    }
  ]
}

라벨링 데이터 포맷 — 필드별로 매핑 완료된 상태:

{
  "image_file": "card_001.jpg",
  "fields": {
    "company_name": { "text": "ABC Corporation", "confidence": 0.97 },
    "person_name":  { "text": "김민우", "confidence": 0.95 }
  }
}

3-3. 분류기 + 파이프라인 작성

분류기는 규칙 기반(regex + 키워드 매칭)으로 먼저 만들었다. 2-pass 방식으로 동작한다:

  1. 1st pass — 확실한 패턴부터 분류 (이메일 regex, 010 휴대폰)
  2. 2nd pass — 나머지 블록을 문맥 참조해서 분류 (유선번호가 남았는데 이미 phone이 있으면 fax로 추정)

단위 테스트를 작성해서 분류기만 단독으로 검증했다:

$ python -m tests.test_classifier
모든 테스트 통과!

3-4. PaddleOCR 3.x API 대응

처음에 PaddleOCR 2.x 기준으로 코드를 작성했는데, 실제 설치된 버전이 3.x라 API가 달랐다. ocr.ocr() 대신 ocr.predict()를 쓰고, 결과 구조도 Result 객체의 .json 속성 안에 res.rec_texts, res.rec_scores, res.rec_polys로 바뀌어 있었다. 이 부분을 수정했다.

4. 트러블슈팅 기록

PROBLEM

명함 사진에서 텍스트가 하나도 검출되지 않음

파이프라인을 실행했는데 결과가 완전히 비어 있었다:

$ py -m src.pipeline.extract_pipeline test.jpg

========================================
  명함 인식 결과
========================================
  오류: 텍스트를 감지하지 못했습니다.
========================================
원인 파악 과정

단계별로 역추적

PaddleOCR의 raw 결과를 직접 뜯어봤다. res.json을 찍어보니 rec_texts: [], rec_polys: []으로 비어 있었다.

한 단계 더 올라가서 텍스트 검출(detection) 결과를 확인했다:

dt_polys count: 0          ← 텍스트 영역 자체를 못 찾음
text_det_params: {
    'limit_side_len': 64,  ← 이게 문제
    'limit_type': 'min',
    'max_side_limit': 4000,
    'box_thresh': 0.6,
    ...
}

원본 이미지 크기가 3000 × 4000px이었다. PaddleOCR 3.x의 텍스트 검출기가 내부적으로 이미지를 축소하는 과정에서 limit_side_len: 64 설정 때문에 텍스트가 사라져버린 것이다.

검증

Pillow로 이미지를 960×1280으로 줄인 다음 OCR을 돌려봤다:

Resized: (960, 1280)
dt_polys count: 8          ← 8개 텍스트 영역 검출
rec_texts: [

    'Fax 053-289-4021Mobile 010-5140-3662'
]
해결

OCR 엔진에 자동 리사이즈 전처리 추가

paddle_ocr_engine.py에 _preprocess_image() 메서드를 추가했다. 이미지의 긴 변이 1280px을 초과하면 자동으로 리사이즈한 뒤 OCR을 돌린다. 

MAX_SIDE = 1280

def _preprocess_image(self, image_path: str) -> str:
    img = Image.open(image_path)
    max_dim = max(img.size)

    if max_dim <= MAX_SIDE:
        return image_path

    ratio = MAX_SIDE / max_dim
    new_size = (int(img.size[0] * ratio), int(img.size[1] * ratio))
    img_resized = img.resize(new_size, Image.LANCZOS)

    tmp = tempfile.NamedTemporaryFile(suffix=".jpg", delete=False)
    img_resized.save(tmp.name, quality=95)
    return tmp.name

추가로 use_doc_orientation_classify=False, use_doc_unwarping=False도 설정했다.

5. 현재 상태

리사이즈 전처리를 넣고 다시 파이프라인을 돌린 결과:

========================================
  명함 인식 결과
========================================
  직책: 선임연구원
  이메일: Tel 053-812-0274 E-mail jcho@gitc.or.kr
  팩스번호: Fax 053-289-4021Mobile 010-5140-3662
========================================

OCR + 파이프라인은 동작하게 됐지만, 분류기에서 문제가 남아 있다.

블록별 분류 상세:

OCR 텍스트분류 결과정답문제
Gi기C unknown - 로고 텍스트, 무시해도 됨
(재)경북IT융합산업기술원 unknown company_name (재) 패턴이 회사 키워드에 없음
바이오헬스연구본부 로봇헬스케어센터 unknown 부서명 본부, 센터 등 부서 패턴 없음
조 중재 unknown person_name 이름 사이에 공백이 있어 한글 2~4자 패턴 불일치
선임연구원 job_title ✓ job_title -
Tel 053-812-0274 E-mail jcho@gitc.or.kr email phone + email 한 줄에 전화+이메일이 합쳐져 있음
Fax 053-289-4021Mobile 010-5140-3662 fax_number fax + phone 한 줄에 팩스+휴대폰이 합쳐져 있음

OCR이 동작 안 하는 문제는 해결했지만, 분류기가 아직 실제 명함 데이터를 제대로 처리하지 못하는 상태. 특히 한 줄에 여러 정보가 합쳐진 경우와 한국어 특유의 패턴을 커버하지 못하고 있다.
